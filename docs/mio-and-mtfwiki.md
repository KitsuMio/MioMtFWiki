# MioMtFWiki 与 mtf.wiki 的关系说明

> 常有朋友问起本站和 mtf.wiki（MtF Wiki）的区别，这里做一个完整说明，方便大家根据自己的需求选择合适的资源。

## 一句话总结

- **MioMtFWiki**：紧跟社区风向、结论优先、给出可执行方案的"行动指南"。
- **mtf.wiki**：体量庞大、引用规范、力求穷尽的"知识基建"。

两站不是竞争关系，而是解决不同性质的问题。

---

## 一、先说清楚 mtf.wiki 是什么

在写对比之前，有必要先理解 mtf.wiki 的体量：它由 Project Trans 团队维护，已有近百名 GitHub 贡献者参与，运营多年。目录覆盖精神科诊断、HRT 药物与医疗资源（细到按省份/医院/具体接诊医生分类）、SRS/FFS 全流程、隆乳、嗓音训练、法律法规、证件变更指引等几乎跨性别生活的每个环节，每条结论大多附学术引用或法规来源。

比如《精神科医疗资源综述》一上来就是 ICD-11、ICD-10、DSM-5、DSM-4、CCMD-3 五套诊断标准的完整对照表，连"已过时但国内仍在使用"的旧标准都保留，标注清楚但不删除。

这种规模和严谨度，本质上是大型协作项目的产物：贡献者多、需要统一规范、追求可考证、力求覆盖周全。这是它的优势，也决定了它的性质——**它在做的是知识基建，不是行动指南**。

## 二、定位理念对比

**MioMtFWiki**
- 结论先行，第一屏就是答案
- 紧跟社区风向：内容来自社区正在问的真实问题
- 持续更新：跟着社区正在发生的讨论走
- 目标：让正确答案比群聊里的错误信息传播得更快
- 是"遇到问题第一个该打开的链接"

**mtf.wiki**
- 体量庞大、贡献者众多，追求穷尽覆盖
- 严谨保守：结论大多有出处，旧标准不删除只标注过时
- 是不断扩张的资料库，定位更接近百科或档案馆

## 三、HRT 内容深度对比

这是两站在最核心主题上的关键区别，也是最能体现差异的地方。

### 1. 页面体量

**MioMtFWiki** 只有 2 篇 HRT 相关页面：
- 《HRT 入门》——原理 + 常用药剂量区间 + 注意事项 + 常见误区
- 《简单明确的用药方案》——4 套现成方案（A/B/C/D），照着吃就行

**mtf.wiki** 的 HRT 板块体量大得多：
- 《药物／HRT》：综述、监测、风险、雌激素药物（5 个剂型子页）、抗雄激素药物（5 种药物子页）、SERMs（1 篇）
- 《HRT 医疗资源》：按国家/省份/医院/具体医生分类的资源列表，数十篇
- 单药页面本身也写得很长，比如色普龙一篇就包含 4 种商品对比、用法用量、6 类副作用、化学成分信息

### 2. 怎么吃——核心差异

**Mio**：直接给方案。"方案 A：色普龙 10~12.5mg + 雌二醇凝胶 1~2 计量尺，副作用是……"一句话能看完一套方案，四套方案覆盖大部分常见组合，还告诉你"对 CPA 敏感选哪个"。

**mtf.wiki**：《HRT 综述》详细讲了三类药物（雌激素/孕激素/抗雄）的作用机制、各给药途径优缺点对比、等效剂量换算表，但**没有给出"该选哪个组合"的结论**——比如直接写"螺内酯略不推荐""比卡鲁胺不推荐常规应用"，却不说替代方案具体怎么搭配着吃。读者得自己从信息里拼。

### 3. 单药页面——信息量大，但可执行的那句话被埋住了

以色普龙（CPA）为例，mtf.wiki 那一页包含：
- 4 种不同产地/厂商商品对比（土色、德色、新色、国色），附规格、批文号、包装图、说明书原文链接
- 用法用量（其实就一句话：10~12.5mg/天，维持期可降到 5~6.25mg/天）
- 6 类副作用，每条配文献编号（脑膜瘤、抑郁、高血压高血糖、血栓、泌乳素、肝毒性）
- 化学成分信息（分子式、CAS 号）

真正"该吃多少"的那句话，被包在一整套产品志和文献综述里，新人要先穿过商品辨认和成分表才能找到。

**Mio** 把同一颗药直接放进"和谁搭配吃、吃多少、要注意什么"的场景里，一段话讲完。

### 4. 风险信息——mtf.wiki 详尽到药理学层级

mtf.wiki《用药风险》页面包含：
- 完整的药物不良反应系统器官分类表（按"非常常见/常见/不常见/罕见/非常罕见"分级）
- 11 种不同雌激素类型和剂量的血栓相对风险率对照表
- CYP3A4 肝酶代谢通路表，列出每种 GAHT 药物依赖或影响哪种酶
- GAHT 药物之间、与精神科药物之间的相互作用清单
- 附录：哪些精神科药物会诱导/抑制 CYP3A4

这是非常扎实的药理学参考资料，但对一个刚开始考虑 HRT 的人来说，信息密度远超过"我现在需要知道什么"。

**Mio** 的风险提示更克制：每个方案下面就写这套组合"可能导致XX"的几条要点，看完就懂，不需要先理解肝酶代谢通路。

### 5. 检查与监测

mtf.wiki 单独有一篇《治疗期间的监测》，详细到检查项目、频次建议（按 SOC-8 等指南）。

Mio 在 HRT 入门页末尾用"风险控制"一小节带过：定期查肝肾功能、血钾，出现血栓征兆立刻就医——给的是"做这几件事就够了"的最小集合，不是完整的监测方案。

### 6. 医疗资源

mtf.wiki 有整整一个板块按省份、医院、具体接诊医生列资源，数量庞大。

Mio 目前没有对应内容——这类信息变动快、地域性强，属于会被"独立成单独篇幅"的低频内容，暂未收录。

### HRT 对比小结

|维度|MioMtFWiki|mtf.wiki|
|---|---|---|
|页面数量|2 篇|十几篇（HRT 板块）+ 数十篇（医疗资源）|
|核心问题|"我该怎么吃"|"这个药/这件事是什么"|
|给出方案|是，4 套现成组合|否，给信息不给结论|
|引用文献|基本不引用，靠社区经验总结|几乎每句话都有文献或说明书出处|
|适合场景|现在就要决定怎么开始|想查清楚某个细节、核对剂量依据|

## 四、内容取舍逻辑

- **MioMtFWiki** 主动省略低频内容（各地具体医生信息、各国证件细则、商品厂商对比等），优先保证高频问题"看五分钟就知道下一步"。低频内容如果后续收录，会**独立成单独篇幅**，不塞进核心页面稀释简洁性。
- **mtf.wiki** 作为多人协作的大型项目，天然倾向于把所有相关主题收全——这是它的体量和定位决定的，不是取舍失误。

简单说：Mio 的简洁是**编辑取舍**的结果；mtf.wiki 的庞大是**项目性质**的结果。

## 五、我们为什么要做 MioMtFWiki

不是因为 mtf.wiki 不够好，而是因为它解决的是另一个问题。一个刚怀疑自己是 MtF、此刻就想知道"我现在该做什么"的人，打开一份五套诊断标准的对照表，或是一篇某种药物的详尽百科词条，得到的是大量准确但需要自己消化、自己拼装的信息——这一步，恰恰是新人最先撞上、也最容易在群聊谣言里翻车的一步。

MioMtFWiki 想补的就是这一步：用结论先行、可执行的方式，先接住"现在怎么办"，再引导有需要的人去 mtf.wiki 深挖原理、查证出处、找具体资源。

## 六、谁适合用哪个

- **刚接触、急需指导、想知道"我现在该怎么办"** → 先看 MioMtFWiki
- **想系统学习原理、查证文献、对比不同厂商商品、找具体医院/医生资源** → 参考 mtf.wiki
- **两者结合使用** 效果最好：Mio 给出可执行的起点，mtf.wiki 提供背景、出处和细节资源，最终仍建议结合专业医疗指导

## 我们的态度

我们很尊重 mtf.wiki 团队多年来投入的精力，这种规模的知识基建对社区是不可替代的资产。两站解决的问题不同——一个接住"现在怎么办"，一个支撑"我想查清楚"——所以更多是互补而非竞争。

我们欢迎大家在两站之间自由切换、交叉参考，也欢迎向我们反馈问题或贡献内容。
